# Residential Moving — Estimator Intake Checklist

Ask for whatever the job spec is still missing, in roughly this order. Never
re-ask for a field the document intake already filled with a non-null value
— confirm it instead if it seems inconsistent with something the caller just
said.

1. Origin: city, state, unit type, bedrooms, floor, elevator access, stair
   count, long-carry distance.
2. Destination: same fields.
3. Distance in miles, if known; otherwise leave `unknown` — do not guess.
4. Move date: preferred date and how many days of flexibility.
5. Inventory: room-by-room major items, box estimate, any special/fragile
   items.
6. Services requested: packing, disassembly/reassembly, storage, valuation
   or insurance preference.

For every field, distinguish "the user doesn't know yet" (`unknown`) from
"this doesn't apply to their move" (`not_applicable`) — never leave a field
ambiguous between the two.

Before submitting the draft: read back a full summary and ask the user to
confirm or correct it. Submitting the draft is not the same as confirming it
— the user still confirms in the dashboard before any calls are made.
